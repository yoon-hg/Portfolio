🏗️Producer / Worker 이중 구조
PRODUCER 외부 API 요청 수신 → Work 객체 생성 (workKey + request + status:NEW) → workDB 저장

WORKER BatchScheduler (1초마다) → workDB 조회 (status=NEW) → SELECT FOR UPDATE
→ status = DOING 선점 → BatchService.batchFunction() → status = DONE
// BatchFunctionMap - 동적 함수 실행
⚙️함수형 인터페이스 Map 구조
  • BATCH_FUNCTION_MAP에 key(workKey) → Function 매핑으로 등록
  • Work 객체의 workKey로 실행할 함수를 동적으로 조회 및 실행
  • 새로운 기능 추가 시 BatchFunctionMap에 함수만 등록하면 자동 처리
  • 예시: "getNotice" → getNotice 함수, "countNoticeView" → countView 함수
// Work 상태 관리 & 동시성 제어
📊Work 상태값
상태코드의미
NEW0처리 대기
DOING1처리 중
DONE2처리 완료
ERROR3에러
🔒동시성 제어
  • SELECT FOR UPDATE로 동일 작업 중복 선점 방지
  • 다른 Worker가 이미 DOING 상태로 변경했다면 SQLException 발생
  • 멀티 Pod 환경에서도 순차 처리 보장
  • @Async + @Scheduled 조합으로 비동기 처리
